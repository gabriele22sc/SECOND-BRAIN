[[FIREBASE]]


Un database è un insieme strutturato di informazioni gestito e organizzato dall'applicazione **DBMS(DataBase Management System)**. Un #DBMS accede facilmente alla rappresentazione logica dei dati. Ogni applicazione usata per la realizzazione di un database memorizza i dati in un formato proprietario. Nei database odierni la rappresentazione logica avviene tramite un #MODELLO-RELAZIONALE, in esso i dati sono organizzati in una struttura a #TABELLA. L'elemento he forma la tabella è il #RECORD, un insieme di dati riguardanti un oggetto trattati in una singola entità. Ogni record è definito da una serie di "attributi": nella struttura dei database essi sono detti "campi"(record = righe, campi = colonne). Altra caratteristica fondamentale dei database relazionali è che possono essere interrogati tramite un linguaggio standard: #SQL.
Quando facciamo comunicare le applicazioni web con un database abbiamo difronte due soggetti che parlano "lingue diverse", e per farli comunicare abbiamo bisogno degli "interpreti", che sono i DRIVER o DATA PROVIDER. I database comunicano grazie alla tecnologia **ADO(ActiveX Data Objects)**, che supporta due interfacce: #OLE-DB e #ODBC. L'accesso della prima è diretto e più veloce mentre ODBC si contrappone tra la nostra applicazione e il data provider OLE-DB.
